## 2026-06-04 16:10 CT - Angora combo CSS first paint

- Fixed a first-paint race where `RuntimeService` could ask Angora to generate authored component classes before `AngoraCombosService` had applied the latest `ConfigStoreService` combo payload, so combo-only classes such as `sectionTitle` were filtered and the Eros draft could reveal with stale title color.
- `AngoraCombosService` now synchronizes draft combos from the store before explicit class updates, splits authored class strings into individual tokens, and checks rendered combo `__COM_*` rules before reporting CSS-ready.
- `RuntimeService` now re-samples rendered DOM classes inside loading-curtain readiness, requires stable ready passes before starting the hide path, and keeps timeout behavior resilient with a timer fallback around frame waits. Combo readiness checks critical color/text markers such as `__COM_sectionTitle-titleColor` and the matching elements' computed token color, because partial combo rules can exist before the final rendered color is active.
- Production readiness timeout for Angora combo CSS increased to 20 seconds after Eros production showed the final `sectionTitle` color could arrive after the old 4-second budget.
- Added final rendered-text color gates for critical draft text combos (`sectionTitle`, `sectionSubtitle`, `heroCaption`) in both `RuntimeService` and `LoadingCurtainService` so no early-exit path can start the boot curtain fade while visible text still computes stale theme colors.
- `LoadingCurtainService` resolves expected critical text colors from the draft theme payload before falling back to CSS variables, covering production cases where `--ank-titleColor` itself is briefly stale.
- Validation passed: focused `AngoraCombosService` + `RuntimeService` Karma suites 36/36 with Edge as `ChromeHeadless`, `npm run build`, local SSR first-paint QA for `erosbarajas.com` on desktop/mobile, and an extended local desktop probe that kept the curtain up until the final title color was present.
